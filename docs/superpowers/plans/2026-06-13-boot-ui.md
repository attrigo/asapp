# Boot-UI Developer Console Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add the [boot-ui](https://github.com/jdubois/boot-ui) embedded developer console to the three API services (`asapp-authentication-service`, `asapp-tasks-service`, `asapp-users-service`) as a dev-only tool, reachable from local runs and the docker dev stack, with no security-code changes.

**Architecture:** boot-ui is a runtime/optional Maven dependency (mirroring `spring-boot-devtools`). It auto-activates under the `dev` profile and force-disables under `prod`; it registers its own highest-precedence `/bootui/**` permit-all security chain and reads Actuator data in-process, so no `SecurityConfiguration` or management-port changes are needed. Work is sequenced **pilot-first** on tasks-service to verify boot-ui `1.4.0` runs on this repo's Spring Boot **4.0.5** before touching the other two services. The existing `DevToolingLockdownIT` (per service) and `asapp-tooling-exposure.jmx` (JMeter) regression surfaces are extended to cover `/bootui`.

**Tech Stack:** Spring Boot 4.0.5 · Java 25 · Maven (multi-module reactor) · `com.julien-dubois.bootui:bootui-spring-boot-starter:1.4.0` · JUnit 5 + RestTestClient + Testcontainers (ITs) · Apache JMeter 5.6.3 (exposure plan).

**Reference spec:** `docs/superpowers/specs/2026-06-13-boot-ui-design.md`

---

## Conventions for this plan

- **Working dir:** repo root `C:\dev\repos\ttrigo\asapp` unless a step says otherwise. Commands are shown for the Bash tool (Git Bash); the session shell is PowerShell — paths use forward slashes either way.
- **Reactor module selector:** `-pl :<artifactId> -am` builds the module plus its upstream deps.
- **Fast vs. slow commands:** `mvn ... -DskipTests` and `mvn spotless:apply` are fast (run freely). `mvn ... verify` and any `*IT` run integration tests via Testcontainers (needs Docker) and are **slow** — per project policy, confirm with the developer before running.
- **Pre-commit hook** runs on every `git commit`: Spotless `check`, LF-line-ending check, and Conventional-Commit message validation. After editing any **Java** file, run `mvn spotless:apply` before committing. POM/`.properties`/`.jmx`/`.md` files are not Spotless-formatted but must keep LF endings.
- **POM ordering** (`.claude/rules/maven.md`): within each comment-delimited group, `<properties>` sort by property name; `<dependencies>`/`<dependencyManagement>` sort by `<groupId>` then `<artifactId>`.

        "management.info.env.enabled=false",
        "springdoc.api-docs.enabled=false",
        "springdoc.swagger-ui.enabled=false" })
