[[overview]]
= Overview

REST API documentation for the ASAPP Tasks Service.
This service manages task lifecycle operations including creation, retrieval, update, and deletion.

All endpoints require Bearer token authentication.

[[overview-http-verbs]]
== HTTP verbs

|===
| Verb | Usage

| `GET`
| Used to retrieve a resource

| `POST`
| Used to create a new resource

| `PUT`
| Used to update an existing resource

| `DELETE`
| Used to delete an existing resource
|===

[[overview-http-status-codes]]
== HTTP status codes

|===
| Status code | Usage

| `200 OK`
| The request completed successfully

| `201 Created`
| A new resource has been created successfully

| `204 No Content`
| The request completed successfully and there is no content to return

| `400 Bad Request`
| The request was malformed. The response body will include an error providing further information

| `401 Unauthorized`
| The request requires authentication. A valid Bearer token must be provided

| `404 Not Found`
| The requested resource did not exist

| `500 Internal Server Error`
| An unexpected server-side failure occurred
|===

[[resources-tasks-get]]
=== Get Tasks

Retrieves tasks from the system. By default, returns all registered tasks; when the `ids` query parameter is supplied, returns only the tasks whose identifiers are in the list.
Missing identifiers are silently omitted; the response order is not guaranteed.
Duplicate identifiers are deduplicated server-side. The `ids` list, when present, must contain between 1 and 50 identifiers.
If no tasks match, an empty array is returned.

operation::get-tasks[snippets='query-parameters,http-request,response-fields,curl-request,http-response']

[[tasks-errors]]
=== Errors

All error responses follow the https://www.rfc-editor.org/rfc/rfc7807[RFC 7807 Problem Details] format.

[[tasks-errors-400-path]]
==== Validation Failure - Path Variable (400)

Returned when a path variable cannot be converted to its expected type (e.g., a malformed UUID).
The response contains only `title`, `status`, and `detail`; no `fieldErrors` array is produced for type-conversion failures.

operation::error-path-variable-validation-failure[snippets='http-response,response-fields']

[[tasks-errors-400-param]]
==== Validation Failure - Request Parameter (400)

Returned when a query parameter fails validation (empty, out of range, or
malformed). `fieldErrors[].field` is the parameter name.

operation::error-request-param-validation-failure[snippets='http-response,response-fields']

[[tasks-errors-400-body]]
==== Validation Failure - Request Body (400)

Returned when the request body contains invalid or missing fields.
The `fieldErrors` array describes each validation failure.

operation::error-request-body-validation-failure[snippets='http-response,response-fields']

[[tasks-errors-401]]
==== Unauthorized (401)

operation::error-unauthorized[snippets='http-response']

==== Not Found (404)

operation::error-not-found[snippets='http-response']
